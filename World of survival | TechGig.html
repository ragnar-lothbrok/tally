World of survival

Time Remaining 4:27:27
There is a plant species (say P) in a desert that has a specific living pattern. A rectangular field of the land is divided into square cells. The land will have R rows and C columns of the cells. Each cell will be surrounded by up-to eight neighbouring cells (note that the cells at the edge of the field will have fewer than eight neighbouring cells). A cell can either be empty (having no plant in it) or living (having a plant in it). A particular cell is said to have N neighbours (1<=N<=8), if exactly N cells in the neighbourhood have living plants.



The plants follow a certain rules for survival. The rules are as follows:
1) If a cell with plant has S1 to S2 neighbors (1 <= S1 <= S2 <= 8), it survives to the next generation. i.e. the plant dies if it has less than S1 neighbors or more than S2 neighbors.

2) If an empty cell has B1 to B2 neighbors (1 <= B1 <= B2 <= 8), a new plant takes birth in that cell in the next generation.

For example, consider a 6X6 field. S1=2, S2=3, B1=B2=3. The initial population of the field is as follows:

                                                                          



The white cells are empty and green cells are occupied. In the following diagram, cells are marked with letters S and B, using the rules of survival. S means live cells that would survive to the next generation, and B means empty cells that would have plants in the next generation.

You have to simulate the survival pattern of the plant given the initial conditions of the desert for given number of generations. Input/Output Specifications Input specification: 
 The input will have two arrays, as follows 
1) An integer array: 
{R,C,S1,S2,B1,B2,G} 
Where R and C specify size of the field in rows and columns respectively, With 1 <= R <= 2000 and 1 <= C <= 2000 
S1, S2, B1 and B2 are integers as explained in the survival rules. 
G is the number of generations for which the simulation has to be run. 

2) An integer array specifying initial state of the field: 
Size of the array will be R*C. The data will be expressed in the array as follows: 
{ E11,E12, … E1C,E21, ...,E2C, … ,ER1, …,ERC} 
where Eij is cell in the field at ith row and jth column. 
Eij can have value 0 or 1. 0 corresponds to empty cell and 1 corresponds to living cell. 

Output Specification: Output will be an integer array specifying state of the field at the end of the simulation in the same manner as in input.

Example 2:

Input2:{0,1,0,0,0,1,1,0,1,0,1,0}
Output:{0,0,1,1,0,1,0,1,0,0,0,0}

Problem Statement

A rice seller has n kg of rice. There is price list for n Kg and less than n Kg of rice. For example let say a seller has 8 Kg of rice and price list is as follows:

Weight(Kg)12345678
price(Rs.)158910171720

Now the question is to how he should sell the n Kg of rice so that he got the maximum benefit. You have to answer the maximum benefit.

For above example if he sells the 8 Kg as whole he would earn Rs. 20 if he sells 1 Kg to 8 different people he would earn 8*1=Rs. 8. But if he sells it as 2 Kg and 6 Kg he would earn 5+17=22 which is the maximum he can get in the above example.

So you are given n Kg and price list of 1 to n Kg and you have to answer what is the maximum benefit the seller can get by any partition of n Kg.

Input Specifications

1) n Kg
2) Array of length n containing the price list from 1 to n kg.

Output Specifications

maximum benefit which can be obtained by any partition of n Kg rice.
